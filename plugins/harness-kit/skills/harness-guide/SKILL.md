# Harness Design ガイド

Anthropic、Manus、LangChain、OpenAI Codex、Stripe Minionsの研究・実践から抽出したハーネス設計のベストプラクティス。

## 核心原則

### Agent = Model + Harness
モデルは制御できないがハーネスは制御できる。LangChainはモデルを変えずハーネスだけで+13.7pt改善した。

### Build to Delete
手書きしたロジックはモデル改善で負債になる。Manusは6ヶ月で5回書き直した。モジュール性を保ち、陳腐化に備えて設計する。

## Generator-Evaluatorパターン

生成と評価を分離する。自己評価バイアス（エージェントが自分の出力を称賛する問題）を構造的に解決。

**分離すべきサイン:**
- エージェントが自分の出力を「素晴らしい」と評価するが実際は凡庸
- 品質改善のためのイテレーションが機能しない
- 特定の品質基準を一貫して満たせない

**使い方:** `evaluator` エージェントと `planner` エージェントがこのプラグインに含まれている。

```
1. 「plannerエージェントで仕様を作成して: {ブリーフ}」
2. 仕様を確認・修正
3. 「この仕様に基づいて実装して」
4. 区切りの良いところで「evaluatorエージェントで品質チェックして」
5. FAIL → 修正 → 再評価のループ
```

## セッション間永続化

長時間タスクでは外部アーティファクトがエージェントの記憶になる。

### 必須アーティファクト

| ファイル | 目的 | ポイント |
|---|---|---|
| `features.json` | 機能リストと完了状態 | JSONはMarkdownよりLLMが壊しにくい |
| `claude-progress.txt` | 活動・決定の逐次ログ | Append-only |
| gitリポジトリ | 変更追跡・巻き戻し | 機能ごとにコミット |

### セッション初期化プロトコル

新しいセッション（またはコンテキストリセット後）の立ち上がり手順:

1. `pwd` で作業ディレクトリ確認
2. `claude-progress.txt` と `git log --oneline -20` を読む
3. `features.json` で次の優先タスク（status: "failing"）を特定
4. 開発サーバー起動
5. 既存機能の動作確認（リグレッション検出）
6. 新しい作業を開始

## コンテキスト管理

### コンテキストリセット > サマリゼーション

コンテキストを完全にクリアして新しいエージェントを起動する方が、その場で要約するより効果的（Anthropicの研究）。外部アーティファクトから再構築する。

### コンテキストエンジニアリング5原則（Manus）

1. **ツールは削除せずマスクする**: 定義変更はKVキャッシュを無効化する
2. **ファイルシステムを外部メモリに**: プラン・進捗・中間結果はコンテキスト外に保存
3. **タスクの反復記述**: 50+ツールコール後、目的を見失う。todo.mdを継続的に書き直す
4. **エラーを保持する**: 失敗・スタックトレースは暗黙の学習信号
5. **可逆的な圧縮**: URLがあればページ内容は削除可能、パスがあればファイル本文は省略可能

## 避けるべき失敗パターン

1. **失敗を理解する前の過剰設計** — 最小構成から始め、実際の失敗を観察してから改善
2. **コンテキストの洪水** — 外科的なコンテキスト選択が包括的ダンプに勝る
3. **早すぎるマルチエージェント化** — シングルエージェントが失敗する証拠がある場合のみ追加
4. **完了前ゲートの欠如** — モデルはタスクを早期に完了宣言する
5. **エラーメッセージの削除** — フィードバックループが壊れる
6. **静的なハーネス** — モデルの改善に合わせて前提を更新する
7. **リトライは最大2回** — 3回目は無駄（Stripeの実証）

## 決定論的ノードとエージェントノードの交互配置（Stripe型）

```
Agent:        エージェントがコード作成
Deterministic: リンター実行（スキップ不可）
Agent:        エージェントがリンターエラー修正
Deterministic: テスト実行
Agent:        エージェントがテスト修正（最大2回）
Deterministic: まだ失敗 → 人間にエスカレーション
```

5つの本番ハーネスチームすべてが、エージェントの自由を拡張ではなく制限することで成功した。
