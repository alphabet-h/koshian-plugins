# NFD ヘルプ — 状態診断と次のステップ案内

NFDワークフローの現在の状態を自動診断し、ユーザーに次にすべきことを案内する。

## 手順

### Step 1: NFD基盤の状態を確認

以下の項目を順にチェックし、状態を判定する:

1. **NFDディレクトリの存在**: `.nfd/` または Glob `**/nfd/config.md` で探索
2. **config.md の存在と設定内容**: YAMLフロントマターを読み込み、データソースの設定状況を確認
3. **crystallization-log.md の存在と実行履歴**: 結晶化が過去に実行されたか
4. **traces/ ディレクトリの内容**: 手動トレースが蓄積されているか
5. **crystals/ ディレクトリの内容**: 結晶化レポートが存在するか
6. **独立Gitリポジトリの状態**: `.nfd/` 内に `.git` が存在するか

### Step 2: 状態に応じた案内を出力

診断結果に基づいて、以下のパターンから適切な案内を選択する:

---

#### パターン A: NFD未導入（.nfd/ が存在しない）

```
## NFD 状態: 未導入

このプロジェクトにはまだNFD基盤が構築されていません。

### 次のステップ
1. `/nfd-init` を実行して基盤を構築してください
2. 生成された `.nfd/config.md` でデータソースを設定してください

### NFDとは？
AIエージェントを「育てる」開発手法です。日常の開発経験を定期的に
結晶化（パターン抽出）し、ルールやスキルに昇華させることで、
AIの応答品質を継続的に改善します。

詳しくは: documents/nfd-methodology.md（プラグイン内のリファレンス）
論文: https://arxiv.org/abs/2603.10808
```

---

#### パターン B: 基盤構築済み・未設定（config.md のデータソースがデフォルトのまま）

```
## NFD 状態: 基盤構築済み（設定推奨）

NFD基盤は構築されていますが、データソースがデフォルト設定のままです。

### 現在有効なデータソース
- ✅ auto memory（MEMORY.md + memory/）
- ✅ セッションログ（JSONL）
- ✅ NFDトレース（.nfd/traces/）
- ❌ カスタムソース（未設定）
- ❌ claude-mem MCP（無効）

### 次のステップ
デフォルト設定でもすぐに結晶化を実行できます:
1. **新しいターミナル**で `claude` を起動（認知干渉防止のため）
2. `/crystallize` を実行

プロジェクト固有のデータソースがあれば `.nfd/config.md` に追加できます:
例: 設計判断ログ、振り返りメモ、開発ノート等
```

---

#### パターン C: 設定済み・結晶化未実行（crystallization-log.md に実行履歴なし）

```
## NFD 状態: 設定済み（初回結晶化を推奨）

NFD基盤と設定が完了しています。初回の結晶化を実行しましょう。

### 設定されているデータソース
（config.md の内容を読み取って列挙する）

### 次のステップ
1. **新しいターミナル**で `claude` を起動（サージカルワークスペース）
2. `/crystallize` を実行
3. 結晶化レポートを確認し、有用なパターンをルール/スキルに統合
```

---

#### パターン D: 結晶化実行済み（crystals/ にレポートが存在する）

```
## NFD 状態: 運用中 ✓

### 結晶化の実行履歴
（crystallization-log.md の実行履歴を読み取って表示）

### 最新の結晶化レポート
（最新のcrystals/*.mdファイル名を表示）

### 次のステップ
- **結晶化レポートの確認**: 最新レポートを読んで、統合すべきパターンがないか確認
- **次回の結晶化**: 新しい経験が蓄積されたら、再度 `/crystallize` を実行（週次〜月次が目安）
- **トレースの記録**: 日常の作業中に気づいたことがあれば `.nfd/traces/` に記録

### 運用のヒント
- 結晶化は **別セッション** で実行する（認知干渉の防止）
- パターンをルール/スキルに統合する際は、必ず自分の経験と照合する
- 1件の事例だけからルールを作らない（過学習防止）
```

---

#### パターン E: 結晶化レポートに未統合の推奨アクションがある場合

パターン D に加えて、最新の結晶化レポートを Read し、「推奨アクション一覧」セクションに「ルール化候補」「スキル化候補」が存在するか確認する。存在する場合は以下を追加:

```
### ⚠ 未統合のパターンがあります
最新の結晶化レポート（crystals/YYYY-MM-DD-crystallization.md）に
統合推奨のパターンがあります。確認して統合を検討してください:

（ルール化候補・スキル化候補のパターン名を簡潔に列挙）
```

---

### Step 3: クイックリファレンスを表示

どのパターンでも、最後に以下のクイックリファレンスを表示する:

```
### NFD クイックリファレンス

| コマンド/操作 | 説明 |
|--------------|------|
| `/nfd-init` | NFD基盤を構築 |
| `/crystallize` | 経験を結晶化（別セッションで実行推奨） |
| `/nfd-help` | この案内を表示 |
| `.nfd/config.md` | データソースの設定 |
| `.nfd/traces/` | 手動トレースの記録先 |
| `.nfd/crystals/` | 結晶化レポートの出力先 |

### NFDの3層構造
- **憲法層**: CLAUDE.md + .claude/rules/ — 根本原則
- **スキル層**: .claude/skills/ — タスク特化型スキル
- **経験層**: セッションログ + auto memory + トレース — 日々の蓄積
         ↑ /crystallize で経験をスキル層に昇華
```
